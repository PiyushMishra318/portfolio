Features

Everything a Design Toolchain Needs

SVG-Palette-Processor handles the full color extraction and transformation pipeline — so your design tokens are always in sync with your SVG source files.

Fill & Stroke Extraction

Parses every fill and stroke attribute across your SVG tree, normalizing hex, RGB, named colors, and opacity into RGBA objects.

Gradient Stop Parsing

Captures all linearGradient and radialGradient stop colors from <defs>, preserving their IDs and references without mutation.

Color Deduplication

Identifies identical colors appearing across multiple elements and groups them — giving you a clean, deduplicated palette with no duplicates in your token output.

Gradient Rewrite

Automatically replaces flat solid-color fills with generated url(#gradient-*) references and injects the corresponding <defs> block into the updated SVG.

Structured JSON Palette

Outputs a machine-readable svg_json.json with keys for fills, strokes, gradients, and obsoleteGradients — ready for any design token pipeline.

CLI & Library API

Use it as a one-liner CLI command or import processSvg / processSvgFile directly in your Node.js build scripts, test suites, and design automation workflows.

Process

From SVG to Palette in Four Steps

A straightforward pipeline from raw SVG input to structured palette output and gradient-enhanced SVG.

  1. 1

    Provide an SVG

    Point the CLI at any SVG file on disk — local fixture, design export, or icon set.

  2. 2

    Parse & Normalize

    The parser traverses the SVG DOM, extracts all color values, and normalizes them to RGBA objects.

  3. 3

    Deduplicate & Map

    Identical colors are consolidated; each unique color is mapped to a gradient definition ID.

  4. 4

    Write Outputs

    Two files land on disk: an enriched SVG with gradient refs and a JSON palette for your token workflow.

Usage

One Command. Two Outputs.

Install globally or use as a local dev dependency — the API stays the same either way.

  • Specify custom output paths with -o and -j flags
  • Works as a Node.js require — integrate into any build script
  • Node.js 18+ required — zero external parser dependencies
  • Full test suite with fixtures included in the repo
bash
$ npm install -g svg-palette-processor
# run against your SVG
$ svg-processor icons/brand.svg \
  -o updated.svg -j palette.json

# or use the library API
const { processSvgFile } = require ('./src/svg-processor');
const { svg, palette } = processSvgFile( 'brand.svg' );
palette .fills // RGBA array
palette .gradients // gradient map
