How It Works

From URL to report in four steps

Whether you use the web UI, REST API, or CLI sitemap mode — the pipeline is always the same straightforward sequence.

  1. 01

    Enter a URL

    Paste any public URL into the web UI form, send a GET /analyze?url= request, or set TARGET_WEBSITE in your .env for sitemap crawl mode. Authentication is handled by your PAGESPEED_KEY environment variable.

  2. 02

    Parallel strategy execution

    The NestJS service fires both desktop and mobile strategy requests to the PageSpeed Insights API concurrently via Promise.all — cutting total wait time roughly in half. The loading indicator reflects both in-flight requests.

  3. 03

    Data extraction & normalization

    Raw Lighthouse JSON is parsed: overall category scores (0–100), individual lab metrics, and the top-five audit opportunities with their estimated byte/time savings are extracted into a clean, consistent response shape.

  4. 04

    Report rendered or saved

    In UI mode the results are rendered instantly — color-coded scores, metric grid, and opportunity list. In CLI mode a report.json is written to REPORT_PATH. Both paths share the same typed TypeScript service layer.
