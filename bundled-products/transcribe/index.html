Everything you need to transcribe at scale

From a single episode to an entire campaign archive — one command handles it all.

Batch Transcription

Point the CLI at any directory of MP4 files. Every video is processed in sequence, with outputs written next to each source file — no configuration required.

Speaker Diarization

Integrates sherpa-onnx to identify who is speaking at each moment. Outputs speaker-tagged ASS subtitle files for multi-participant sessions.

GPU Accelerated

CUDA support out-of-the-box on Windows via MSVC. Falls back gracefully to CPU-only mode with a single flag — works on any machine.

Multiple Output Formats

Generates .srt for standard subtitles, .txt for plain transcripts, and .ass for speaker-tagged advanced subtitles. All three, simultaneously.

CLI-First Design

Install once, use from anywhere. cd into your episodes folder and run. Built-in help system with per-topic docs — transcribe help profiles.

Model Flexibility

Choose from whisper base (~150 MB) for speed or medium (~1.5 GB) for accuracy. Swap models per-run or set a default in your profile.

Cross-Platform

Full support for Windows 10/11 (CUDA + MSVC), Ubuntu 22.04+, and macOS 12+. Install scripts handle all dependencies for each platform.

ffmpeg Integration

Leverages ffmpeg for audio extraction from MP4. Any ffmpeg-compatible container works — MP4, MKV, MOV, WebM, and more.

From raw MP4 to subtitles in four steps

The workflow is intentionally straightforward. No config files, no web interface — just a terminal and your episode folder.

  1. 01

    Install the CLI

    Clone the repo and run the install script for your platform. The binary is placed in a system location and added to your PATH automatically. Open a new terminal to confirm installation.

    transcribe --version
  2. 02

    Navigate to your episode folder

    The CLI is directory-oriented. The working directory is your source. Every MP4 found there becomes a transcription target — no globbing needed.

    cd ~/recordings/campaign-s2
  3. 03

    Choose your model and run

    Select a Whisper model size and trigger the batch. whisper.cpp extracts audio via ffmpeg, runs inference, and produces timestamped output for every file.

    transcribe run --model medium
  4. 04

    Collect your outputs

    Outputs are written next to each source MP4. You get an SRT subtitle file, a plain TXT transcript, and — if diarization is enabled — an ASS file with speaker labels for each voice.

    episode-01.mp4 → episode-01.srt, .txt, .ass

Get started in minutes

Choose your platform and follow the steps. The installer handles Rust, CUDA detection, and PATH registration.

PowerShell 
git clone https://github.com/piyushmishra318/transcribe.git
cd episode-transcribe
powershell -ExecutionPolicy Bypass -File scripts/install.ps1
Verify 
transcribe --version
transcribe help
Bash 
git clone https://github.com/piyushmishra318/transcribe.git
cd episode-transcribe
chmod +x scripts/install.sh scripts/uninstall.sh
./scripts/install.sh
Verify 
transcribe --version
transcribe help profiles
PowerShell — CPU Only 
powershell -ExecutionPolicy Bypass -File scripts/install.ps1 -CpuOnly
Bash — CPU Only 
./scripts/install.sh --cpu-only
