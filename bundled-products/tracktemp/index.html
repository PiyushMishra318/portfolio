Architecture

How it works

Four components. One HTTP GET. Zero cloud subscriptions.

  1. DHT11 Sensor

    Wired to D4 (GPIO2). The NodeMCU polls it on a configurable interval — default 30 seconds — reading temperature in Celsius, Fahrenheit, and relative humidity with heat-index computed on-device.

  2. NodeMCU ESP8266

    Firmware built with PlatformIO packs all five readings into a query string and fires an HTTP GET. Credentials live in secrets.h — gitignored by default. GitHub Actions CI is included out of the box.

  3. PushingBox Bridge

    ESP8266 can only speak plain HTTP. PushingBox acts as a free HTTP-to-HTTPS relay, forwarding the payload to your Google Apps Script web app endpoint without any TLS overhead on the microcontroller.

  4. Google Sheets

    A two-line Apps Script appends a timestamped row on every request. Your spreadsheet becomes a live sensor log you can chart, filter, share, or export to CSV instantly — no dashboard product required.

Features

Everything you need to ship

Production-quality firmware for a weekend-project price.

Secrets management

WiFi credentials and the PushingBox device ID live in secrets.h, which is gitignored by default. Safe to open-source your entire firmware repo.

PlatformIO build

Dependency-managed via platformio.ini. No manual library installs — PlatformIO fetches DHT and ESP8266WiFi libraries automatically on first build.

WiFi auto-reconnect

The firmware detects dropped connections and retries automatically, keeping the logging loop running unattended for days without intervention.

Five metrics per row

Each Google Sheets row includes timestamp, Celsius, Fahrenheit, relative humidity, heat-index in Celsius, and heat-index in Fahrenheit — everything you need to analyse comfort levels.

Configurable interval

Change the logging cadence in config.h. The DHT11 supports readings as frequent as every 2 seconds — dial it to your project's bandwidth budget.

GitHub Actions CI

A pre-wired .github/workflows config compiles the firmware on every push, catching build regressions before you flash hardware.

Get started

Up and logging in 4 steps

From unboxing to live data in under 20 minutes.

01

Create Google Sheet & Apps Script

Create a new Google Sheet, open Extensions → Apps Script, paste Code.gs from the repo, set your spreadsheet ID, and deploy as a web app (execute as me, access anyone).

02

Set up PushingBox scenario

Create a free PushingBox account, add a CustomURL service pointing at your Apps Script URL, and create a scenario with the five query-string variables. Copy the Device ID.

03

Flash the firmware

Clone the repo, copy secrets.example.h to secrets.h, fill in your WiFi credentials and Device ID, then run pio run -t upload from the firmware directory.

04

Monitor & verify

Open the serial monitor with pio device monitor. Within 30 seconds you should see a 200 OK confirmation and a new row appear in your Google Sheet.
