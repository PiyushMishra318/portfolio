What it does

Four-layer email trust verification

Every inbound message is evaluated against four independent verdict signals. Any single failure triggers an automatic bounce — before the message reaches a recipient.

SPF Verdict

Verifies the sending server is authorised to send on behalf of the domain. Stops spoofed sender addresses from slipping through.

Sender Policy Framework

DKIM Verdict

Validates the cryptographic signature that proves the message body has not been tampered with in transit.

Cryptographic Integrity

Spam Verdict

Uses AWS SES's built-in spam classification. Messages scoring above the threshold are bounced before delivery.

AWS SpamVerdict

Virus Verdict

Scans all attachments and inline content for known malware signatures. Zero-tolerance policy — any detection triggers bounce.

Malware Scan

Serverless. Zero infra overhead.

Runs as an AWS Lambda receipt-rule action. No servers to manage, no polling loop — SES invokes the handler directly on each inbound message.

AWS Lambda Python 3.11+

Execution flow

1 SES receives inbound message and invokes Lambda handler
2 Handler reads spfVerdict, dkimVerdict, spamVerdict, virusVerdict
3 Any FAIL triggers ses:SendBounce and halts rule set
4 All verdicts pass → message accepted, rule processing continues

Bounce vs Accept

Two clean outcomes for every message. No ambiguity, no partial accept.

Accept All four verdicts return PASS. Rule set processing continues normally.
Bounce One or more verdicts return FAIL. ses:SendBounce fires immediately.

How it works

From SES event to bounce in four steps

The Lambda handler is a pure function with no external dependencies beyond the AWS SDK — fast, testable, deterministic.

  1. 1

    Email arrives at SES

    AWS SES receives the inbound message and runs it through initial receipt rules, resolving SPF, DKIM, spam, and virus verdicts automatically.

  2. 2

    Lambda invoked

    The receipt rule triggers the Lambda handler with the full SES event JSON, including the four verdict objects and message metadata.

  3. 3

    Verdicts inspected

    The handler checks each verdict field. A single FAIL on any check is sufficient to reject — the logic is deliberately strict and short-circuit.

  4. 4

    Bounce or accept

    Failure calls ses:SendBounce and stops the rule set immediately. All clear means processing continues and the message reaches its destination.

Under the hood

The core handler — fewer than 20 lines

All the filter logic lives in one pure function. Easy to audit, easy to test, easy to extend.

src/ses_spam_filter.py Python 
import boto3

VERDICT_FIELDS = ["spfVerdict", "dkimVerdict", "spamVerdict", "virusVerdict"]

def should_bounce(receipt: dict) -> bool:
    """Return True if any verdict is FAIL."""
    return any(
        receipt.get(field, {}).get("status") == "FAIL"
        for field in VERDICT_FIELDS
    )

def lambda_handler(event, context):
    record  = event["Records"][0]["ses"]
    mail    = record["mail"]
    receipt = record["receipt"]

    if should_bounce(receipt):
        boto3.client("ses").send_bounce(...)
        return {"disposition": "STOP_RULE_SET"}

    return {"disposition": "CONTINUE"}
